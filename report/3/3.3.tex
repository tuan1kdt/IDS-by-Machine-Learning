\subsection{Feature Engineering}
\label{sec:feat-eng}

The 78 numeric features produced by \textsc{CICFlowMeter} are used as-is in our experiments. Dropping the flow-identifier metadata columns (\texttt{Flow ID}, source/destination IP, \texttt{Timestamp}) leaves a feature matrix $\mathbf{X} \in \mathbb{R}^{N \times 78}$ whose columns span four semantically meaningful groups: flow-level descriptors (duration, byte/packet counts), packet-size statistics (min/max/mean/std), inter-arrival times (IAT mean/std in both directions), and TCP flag counts (SYN, ACK, FIN, RST, PSH, URG).

This ``raw features only'' approach is intentional. As noted in Section~\ref{sec:bg-ml}, tree-based learners are invariant to monotone column rescaling and do not require domain-engineered interaction features to achieve strong performance on tabular data. The only preprocessing applied before the learner is the \texttt{StandardScaler} fitted on the training split, which centres and normalises each column to unit variance. All four candidate models consume this same $\mathbb{R}^{78}$ representation.

Future work could explore derived features such as forward/backward packet-count ratios or Shannon entropy of the packet-length distribution, both of which have shown modest per-class gains on minority classes (\texttt{Bot}, \texttt{C2 beaconing}) in related literature. We leave this to future investigation to keep the current pipeline simple and reproducible.
