\subsection{Training Procedure}
\label{sec:training}

The training procedure follows a strict no-leakage protocol: all data-altering transformations (cleaning, scaling, SMOTE) are fitted exclusively on the training split and applied without re-fitting to the validation and test splits. This eliminates the two most common sources of inflated scores on CIC-IDS2017: scaler fitted on the full dataset (leaks test statistics) and SMOTE applied before splitting (leaks synthetic test points derived from training data).

The offline pipeline proceeds as follows:
\begin{enumerate}
    \item \textbf{Clean}: drop rows containing \texttt{NaN} or $\pm\infty$; remove exact duplicates.
    \item \textbf{Sample}: cap each class at 50,000 rows; merge classes below 500 samples into \texttt{Rare\_Attack}.
    \item \textbf{Split}: 80/20 stratified train/test split (random seed 42).
    \item \textbf{Scale}: fit \texttt{StandardScaler} on training features only; transform both splits.
    \item \textbf{Encode}: fit \texttt{LabelEncoder} on training labels; transform both splits.
    \item \textbf{SMOTE}: apply Proportional SMOTE (Section~\ref{sec:exp-setup}) to the training split only.
    \item \textbf{Train}: fit all four classifiers on the augmented training set with \texttt{class\_weight=balanced}.
    \item \textbf{Evaluate}: score each model on the held-out test set; report Macro-$F_1$, ROC-AUC, MCC, and Balanced Accuracy.
\end{enumerate}

No hyperparameter search beyond the defaults described in Section~\ref{sec:model-design} was performed; the goal is a reproducible, single-seed evaluation that fairly compares the four classifiers under identical conditions.
