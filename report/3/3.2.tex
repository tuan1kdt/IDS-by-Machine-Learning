\subsection{System Architecture Overview}
\label{sec:arch-overview}

The proposed IDS is organised as a staged pipeline in which each component exposes a narrow data contract to the next, enabling offline training and online inference to share identical preprocessing and feature-engineering logic:

\begin{figure}[h]
\centering
\begin{tikzpicture}[
    node distance=0.65cm and 0.85cm,
    every node/.style={font=\footnotesize},
    stage/.style={draw, rounded corners=2pt, minimum width=2.3cm, minimum height=0.9cm, align=center, thick},
    arrow/.style={-Stealth, thick}
]
    \node[stage, fill=blue!8]   (capture)  {Traffic\\Capture};
    \node[stage, fill=blue!8,  right=of capture]  (extract)  {Flow\\Extractor};
    \node[stage, fill=green!10, right=of extract]  (prep)     {Preprocessing\\\& Scaling};
    \node[stage, fill=orange!15, right=of prep]    (model)    {ML Classifier\\(4 models)};
    \node[stage, fill=red!10,  right=of model]    (alert)    {Alert /\\Log Sink};

    \draw[arrow] (capture) -- node[above=-1pt,font=\scriptsize]{pcap} (extract);
    \draw[arrow] (extract) -- node[above=-1pt,font=\scriptsize]{flow row} (prep);
    \draw[arrow] (prep)    -- node[above=-1pt,font=\scriptsize]{$\mathbf{x}\in\mathbb{R}^{78}$} (model);
    \draw[arrow] (model)   -- node[above=-1pt,font=\scriptsize]{$\hat{y}, p$} (alert);
\end{tikzpicture}
\caption{End-to-end IDS pipeline. The green preprocessing and orange model stages are shared between the offline training path and the online inference path.}
\label{fig:pipeline}
\end{figure}

\textbf{Traffic Capture} mirrors raw packets from a SPAN port and writes them to a rolling PCAP buffer.
\textbf{Flow Extractor} (\textsc{CICFlowMeter}~\cite{ref_cicflowmeter}) groups packets into bidirectional flows and emits one 78-feature row per flow.
\textbf{Preprocessing \& Scaling} drops \texttt{NaN}/$\pm\infty$ rows, applies a frozen \texttt{StandardScaler}, and encodes labels.
\textbf{ML Classifier} scores the scaled feature vector and emits a predicted class and optional confidence.
\textbf{Alert / Log Sink} forwards non-\texttt{BENIGN} predictions to a SIEM and logs all predictions for offline drift monitoring.
